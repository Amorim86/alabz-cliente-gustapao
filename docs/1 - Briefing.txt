TAREFA
Criar uma landing page comercial, responsiva e altamente visual para a Big Pão Panificadora e Confeitaria, considerando exclusivamente a unidade Afonso Pena, em São José dos Pinhais — PR.

A página deve apresentar os produtos e serviços sem exibir preços, despertar desejo pelas imagens e conduzir o visitante para quatro ações:

1. Pedir pelo WhatsApp.
2. Ver cardápio.
3. Fazer encomenda.
4. Como chegar.

Antes de editar qualquer arquivo:

1. Inspecione todo o workspace.
2. Identifique a stack, estrutura de pastas, componentes e padrões existentes.
3. Gere um plano de tarefas objetivo.
4. Informe quais arquivos pretende criar ou modificar.
5. Preserve a stack atual e não substitua a arquitetura sem necessidade.
6. Caso exista uma página anterior, crie uma versão temporária ou backup antes de alterar a principal.

COMPORTAMENTO MOBILE

Implementar mobile first.

No celular:

- manter CTA do WhatsApp visível cedo;
- usar header compacto;
- permitir leitura confortável;
- manter corpo de texto com pelo menos 16 px;
- usar botões com altura mínima de 48 px;
- evitar textos longos no hero;
- empilhar corretamente blocos de imagem e texto;
- não usar carrosséis automáticos;
- não esconder informações importantes em hover;
- evitar elementos que cubram conteúdo.

Criar uma barra inferior fixa e discreta com:

- Pedir;
- Cardápio;
- Como chegar.

A barra deve respeitar safe areas e não encobrir o rodapé ou os botões da página.

ARQUITETURA E CÓDIGO

Se o workspace já tiver uma aplicação:

- respeitar a stack;
- reutilizar componentes;
- preservar padrões;
- não instalar bibliotecas sem necessidade;
- não quebrar rotas existentes;
- não apagar conteúdo sem backup;
- manter o código organizado.

Se o workspace estiver vazio:

- criar uma implementação estática leve e organizada;
- preferir HTML semântico, CSS bem estruturado e JavaScript somente quando necessário;
- não adicionar framework pesado para uma landing page simples;
- separar conteúdo, estilos e comportamento de forma clara;
- organizar assets em pasta própria.

Centralizar informações alteráveis em uma configuração ou objeto de dados, incluindo:

- contatos;
- horários;
- endereço;
- links;
- números da prova social;
- textos;
- categorias futuras;
- imagens.

Não espalhar dados críticos por vários arquivos.
